🚀 AEM QA System 운영 가이드

설치 및 설정 운영 및 유지보수 문제 해결

시스템 설치 및 운영을 위한 대화형 가이드

이 가이드는 AEM QA System의 설치부터 운영, 문제 해결까지의 모든 절차를 안내합니다. '설치 및 설정' 섹션의 체크리스트를 따라 진행 상황을 관리하고, 코드 블록의 복사 버튼을 사용하여 명령어를 쉽게 실행하세요.

1. 설치 및 설정 (Installation & Setup)

시스템을 성공적으로 배포하고 실행하기 위한 단계별 절차입니다.

설치 진행률:

🖥️

운영체제

Linux/macOS/Windows

🐍

Python

3.9 이상

💾

하드웨어

16GB+ RAM, 50GB+ Storage

🗃️

데이터베이스

MongoDB 5.0+, ChromaDB 0.4+

1. Python 가상 환경 생성

python -m venv aem_qa_venv
source aem_qa_venv/bin/activate  # Linux/macOS
# aem_qa_venv\\Scripts\\activate  # Windows

2. 필수 라이브러리 설치

pip install -r requirements.txt

3. 데이터베이스 설치 및 실행

MongoDB는 공식 문서를 참고하여 설치하고, ChromaDB는 아래 명령어로 실행합니다.

chroma run --host localhost --port 8000

프로젝트의 `src/config.py` 파일을 열어 환경에 맞게 주요 변수들을 수정하세요.

  • `MONGO_CONNECTION_STRING`
  • `DB_NAME`
  • `CHROMA_HOST` / `CHROMA_PORT`
  • `AEM_HOST`

터미널에서 `src/scripts` 디렉터리로 이동 후 아래 명령어를 실행하고, 메뉴에서 `[1] 컬렉션 생성`을 선택하세요.

python setup_collections.py

2. 운영 및 유지보수 (Operation & Maintenance)

3.1. 시스템 실행 및 중지

Collector App과 메인 시스템은 각각 다른 명령어로 실행합니다.

# Collector App 실행
streamlit run app.py

# AEM QA System (메인) 실행
streamlit run Home.py

3.2. 시스템 모니터링

메인 시스템 사이드바의 `[DB 모니터링]` 메뉴에서 MongoDB와 ChromaDB의 컬렉션, 문서 수, 저장 공간 등의 상태를 실시간으로 확인하세요.

3.3. 데이터 백업 및 복구

`mongodump`를 사용하여 MongoDB를 백업합니다.

mongodump --uri="mongodb://localhost:27017" --db="aem_qa_system" --out="/path/to/backup/dir"

ChromaDB는 파일 시스템 기반이므로 `chroma/` 디렉터리를 직접 압축하여 백업합니다.

3.4. 로그 관리

시스템 실행 시 로그를 파일로 저장하여 문제를 추적할 수 있습니다.

streamlit run Home.py > logs/app.log 2>&1 &

3. 문제 해결 (Troubleshooting)

`ModuleNotFoundError` 발생 시

가상 환경이 활성화되었는지 확인하고, `pip install -r requirements.txt`를 다시 실행하여 모든 라이브러리가 설치되었는지 확인하세요.

AEM 서버 401 (Unauthorized) 오류 발생 시

Collector App에 입력한 사용자 이름과 비밀번호가 올바른지, 해당 계정이 AEM 서버에 접근할 권한이 있는지 확인하세요.

데이터베이스 연결 실패 시

MongoDB와 ChromaDB 서버가 정상적으로 실행 중인지 확인하고, `src/config.py` 파일의 접속 정보가 올바른지 확인하세요. `src/scripts/test_mongo_connection.py` 스크립트로 연결을 테스트할 수 있습니다.
