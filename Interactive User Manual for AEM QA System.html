Part 1: 데이터 수집 (Collector App)

분석에 필요한 원본 데이터를 AEM 서버로부터 수집하여 하나의 패키지 파일(`.zip`)로 만드는 과정입니다.

2.1. Collector App 실행 및 정보 입력

`app.py`를 실행하여 'AEM 콘텐츠 수집 및 패키징 도구'를 엽니다. 다음 정보를 정확하게 입력하세요.

① AEM 인증 정보

  • AEM 사용자 이름
  • AEM 비밀번호

② 페이지 목록 파일

수집할 페이지 경로가 담긴 .txt 파일을 업로드합니다.

/content/project/us/en/page1
/content/project/us/en/page2

2.2. 수집 시작 및 결과 다운로드

모든 정보 입력 후 `[🚀 수집 시작하기]` 버튼을 클릭합니다. 작업이 완료되면 `[📥 (패키지명).zip 다운로드]` 버튼을 클릭하여 결과물을 저장하세요.

📌 중요

이 단계에서 생성된 `.zip` 파일은 Part 2에서 데이터를 시스템에 적재하기 위한 필수 입력 파일입니다.

Part 2: 데이터 분석 및 QA (AEM QA System)

수집된 데이터를 시스템에 적재하고, 버전 간 비교 분석 및 AI 기반 검색을 통해 실제 QA 작업을 수행하는 과정입니다.

3.1. 데이터 적재 및 처리

새로운 콘텐츠 패키지를 받았을 때 수행하는 초기 설정 작업입니다. `Home.py`를 실행하여 메인 시스템에 접속하세요.

  1. 데이터 적재: 사이드바 `[데이터 적재]` 메뉴에서 Part 1에서 생성한 `.zip` 패키지를 업로드합니다.
  2. TM 처리 및 인덱싱: 사이드바 `[TM 처리]` 메뉴에서 언어 쌍을 선택하고 파이프라인을 시작합니다. 이 과정에서 시스템이 자동으로 콘텐츠를 분리하고 AI 검색을 위한 벡터 데이터를 생성합니다.

✅ Tip

이 작업들은 새로운 데이터를 분석할 때 한 번만 수행하면 됩니다. 이후에는 주로 QA 분석가 작업을 사용하게 됩니다.

3.2. QA 대시보드를 이용한 분석

사이드바 `[QA 대시보드]` 메뉴에서 버전 간 콘텐츠 변경 사항을 분석합니다.

  1. 페이지 목록 업로드: 검수할 페이지 목록이 담긴 .csv 파일을 업로드합니다.
  2. 버전 선택: 분석의 기준이 될 소스(Source) 버전과 타겟(Target) 버전을 선택합니다.
  3. 분석 실행: `[🚀 페이지 분석 실행]` 버튼을 클릭하면 테이블에 페이지별 변경 상태, QA 항목 수, 변경률 등이 표시됩니다.
  4. 리포트 다운로드: 분석 결과를 `[📄 Excel 리포트]` 또는 `[🌐 HTML 리포트]`로 다운로드하여 공유할 수 있습니다.

3.3. 고급 검색

사이드바 `[고급 검색]` 메뉴에서 AI 기반 의미 검색을 통해 번역의 일관성을 검증합니다.

  1. 검색 옵션 선택: 대상 언어와 컬렉션(확정 TM, 용어집 등)을 선택합니다.
  2. 검색어 입력: 번역 품질을 확인하고 싶은 원문 텍스트를 입력합니다.
  3. 결과 확인: 시스템이 의미적으로 유사한 번역 사례를 찾아 소스/타겟 텍스트, 출처, 유사도와 함께 보여줍니다.
